# ハナシテ アバター 命名ルール・管理フロー

## ファイル命名ルール

```
{カテゴリコード}_{連番2桁}.png
例: f20_01.png
```

| カテゴリコード | 内容 | 個数 |
|---|---|---|
| `f20` | 女性 20代 | 10個 |
| `f30` | 女性 30代 | 10個 |
| `f40` | 女性 40代 | 10個 |
| `m20` | 男性 20代 | 10個 |
| `m30` | 男性 30代 | 10個 |
| `m40` | 男性 40代 | 10個 |
| `cat` | ねこ | 10個 |
| `dog` | いぬ | 8個 |
| `rabbit` | うさぎ | 4個 |
| `bear` | くま・パンダ | 4個 |
| `other` | その他動物 | 4個 |

**合計: 人物60個 + 動物30個 = 90個**（＋追加分で100個へ拡張予定）

## Next.js / React での呼び出し方

```tsx
// アバターIDをDBに保存（例: "f20_03"）
const avatarId = user.avatarId  // "f20_03"
const [category, number] = avatarId.split('_')  // ["f20", "03"]
const avatarUrl = `/avatars/${category}/${avatarId}.png`

// <img> タグで表示
<img src={avatarUrl} alt="アバター" width={64} height={64} />
```

## 個別却下・再作成フロー

### 手順

1. **却下するファイルを特定する**
   - ファイル名を確認（例: `m20_03.png`）

2. **却下ファイルをバックアップフォルダに移動する**
   ```bash
   mkdir -p public/avatars/_rejected
   mv public/avatars/m20/m20_03.png public/avatars/_rejected/m20_03_rejected.png
   ```

3. **同じファイル名で再生成を依頼する**
   - Manusに「`m20_03.png` を再生成してください。理由: ○○」と伝える
   - 再生成時は必ず `reference.png` を参照スタイルとして使用する

4. **新しいファイルを同じパスに配置する**
   ```bash
   # 再生成されたファイルを正しいパスに配置
   mv new_m20_03.png public/avatars/m20/m20_03.png
   ```

5. **DBやコードのアバターIDは変更不要**（同じファイル名で上書きするため）

## スタイル統一の基準

すべてのアバターは以下の基準を満たすこと：

- **スタイル**: 日本アニメ風フラットイラスト
- **輪郭線**: 太めのダーク輪郭線あり
- **背景**: ベージュピンクの円形背景、白い正方形キャンバス
- **肌色**: 温かみのあるパステルトーン（人物の場合）
- **サイズ**: 2048×2048px（表示時は64〜128pxにリサイズ）
- **テキスト**: なし
- **参照ファイル**: `reference.png`
