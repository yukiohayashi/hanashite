# レイアウト崩れ対策ガイド

## 問題の概要

デプロイ後、管理画面やユーザー画面のサイドバーレイアウトが崩れる現象が頻発しています。
これはNext.js 13のApp Routerを使用したプロジェクトにおける、ビルドキャッシュとスタイルの適用タイミングに関する問題です。

## 原因

### 1. Next.jsのビルドキャッシュ
- `.next`ディレクトリにキャッシュされたビルド成果物が古いスタイル情報を保持
- 新しいコードをデプロイしても、キャッシュされたスタイルが優先される
- 特にTailwind CSSのクラスが正しく適用されない

### 2. 静的アセットのコピー不足
- `npm run build`後、`.next/static`と`public`ディレクトリを`.next/standalone`にコピーする必要がある
- これを怠ると、CSSファイルやその他の静的ファイルが正しく配信されない

### 3. ブラウザキャッシュ
- ユーザー側のブラウザキャッシュが古いスタイルを保持
- 開発環境では問題なくても、本番環境で崩れる原因になる

## 解決方法

### デプロイ時の正しい手順

```bash
# VPSにSSH接続後、以下を実行
cd /home/ubuntu/hanashite
git pull
pm2 stop hanashite
rm -rf .next
npm run build
cp -r .next/static .next/standalone/.next/
cp -r public .next/standalone/
pm2 restart hanashite
```

### 重要なポイント

1. **必ず`.next`ディレクトリを削除する**
   - `rm -rf .next`でキャッシュをクリア
   - これを省略すると古いスタイルが残る

2. **静的ファイルを正しくコピーする**
   - `.next/static`を`.next/standalone/.next/`にコピー
   - `public`を`.next/standalone/`にコピー

3. **PM2を完全に再起動する**
   - `pm2 restart`だけでなく、`pm2 stop`→ビルド→`pm2 restart`の順序を守る


## ユーザー側の対応

レイアウト崩れが発生した場合、以下をユーザーに案内：

1. **ブラウザキャッシュをクリア**
   - Chrome: Cmd+Shift+Delete（Mac）/ Ctrl+Shift+Delete（Windows）
   - Safari: Cmd+Option+E

2. **シークレットウィンドウで確認**
   - キャッシュの影響を受けないため、最新のスタイルが適用される
   - 開発環境ではシークレットウィンドウでの確認を推奨

## 予防策

### 1. デプロイスクリプトの作成
`/home/ubuntu/hanashite/deploy.sh`を作成：

```bash
#!/bin/bash
cd /home/ubuntu/hanashite
git pull
pm2 stop hanashite
rm -rf .next
npm run build
cp -r .next/static .next/standalone/.next/
cp -r public .next/standalone/
pm2 restart hanashite
echo "Deployment completed successfully!"
```

実行権限を付与：
```bash
chmod +x /home/ubuntu/hanashite/deploy.sh
```

使用方法：
```bash
ssh -i ~/.ssh/hanashite.key ubuntu@133.18.125.19 "/home/ubuntu/hanashite/deploy.sh"
```

### 2. CSSの変更時は必ずフルビルド
- Tailwind CSSのクラスを変更した場合
- レイアウトに関するコンポーネントを変更した場合
- 管理画面のサイドバーなど、重要なUIを変更した場合

### 3. デプロイ後の確認手順
1. シークレットウィンドウで主要ページを確認
2. 管理画面のサイドバーが正しく表示されているか確認
3. ユーザー画面のサイドバーが正しく表示されているか確認

## トラブルシューティング

### 問題: デプロイ後もレイアウトが崩れている
**解決策**: フルビルドを実行

### 問題: 一部のページだけ崩れている
**解決策**: 該当ページのキャッシュをクリアし、フルビルドを実行

### 問題: ブラウザキャッシュをクリアしても崩れている
**解決策**: サーバー側のキャッシュが原因。フルビルドを実行

## 注意事項

- **開発環境ではシークレットウィンドウを使用**
  - ユーザーメモリーに記載されている通り、キャッシュは不問
  - 常に最新の状態を確認するため

- **デプロイ前にローカルでビルドテスト**
  ```bash
  npm run build
  npm start
  ```
  - ローカルでビルドエラーがないか確認

- **緊急時の対応**
  - レイアウト崩れが発生したら、すぐにフルビルドを実行
  - ユーザーにはシークレットウィンドウでの確認を案内

## VPS固有の問題

### 問題: CSSファイルが403エラーで配信されない
**原因**: `/home/ubuntu`ディレクトリのパーミッションが750で、Nginxユーザー（www-data）がアクセスできない

**解決策**: 
```bash
sudo chmod 755 /home/ubuntu
```

**詳細説明**:
- Nginxが`/home/ubuntu/hanashite/.next/standalone/.next/static`配下のCSSファイルにアクセスするには、親ディレクトリ（`/home/ubuntu`）に実行権限（x）が必要
- デフォルトで`/home/ubuntu`のパーミッションが750（`drwxr-x---`）になっている場合がある
- これを755（`drwxr-xr-x`）に変更することで、Nginxがファイルにアクセスできるようになる

**確認方法**:
```bash
# パーミッションを確認
ls -ld /home/ubuntu

# CSSファイルが正しく配信されているか確認
curl -I https://dokujo.com/_next/static/chunks/[CSSファイル名].css
```

### Nginx設定の確認

静的ファイルを直接配信する設定が必須です（`/etc/nginx/sites-available/dokujo.com`）:

```nginx
location /_next/static {
    alias /home/ubuntu/hanashite/.next/standalone/.next/static;
    expires 365d;
    access_log off;
}

location /images {
    alias /home/ubuntu/hanashite/.next/standalone/public/images;
    expires 365d;
    access_log off;
}
```

設定変更後は必ず:
```bash
sudo nginx -t
sudo systemctl reload nginx
```

## まとめ

レイアウト崩れの主な原因は以下の3つです：

1. **ビルドキャッシュ** - `.next`ディレクトリを削除してフルビルド
2. **静的ファイルのコピー不足** - `.next/static`と`public`を正しくコピー
3. **パーミッション問題（VPS）** - `/home/ubuntu`のパーミッションを755に設定

デプロイ時は必ず`.next`ディレクトリを削除し、静的ファイルを正しくコピーし、
パーミッションを確認することで、この問題を防ぐことができます。

**重要**: CSS変更時は必ずフルビルドを実行し、パーミッションを確認してください。
