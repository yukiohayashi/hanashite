## ファイル命名ルール

```
{カテゴリコード}_{連番2桁}.png
例: f20_01.png
```

| カテゴリコード | 内容 | 個数 |

## 個別却下・再作成フロー

### 手順

1. **却下するファイルを特定する**
   - ファイル名を確認（例: `m20_03.png`）

2. **却下ファイルをバックアップフォルダに移動する**
   ```bash
   mkdir -p public/images/local-avatars/_rejected
   mv public/images/local-avatars/m20_03.png public/images/local-avatars/_rejected/m20_03_rejected.png
   ```

3. **同じファイル名で再生成を依頼する**
   - Manusに「`m20_03.png` を再生成してください。理由: ○○」と伝える
   - 再生成時は必ず `reference.png` を参照スタイルとして使用する

4. **新しいファイルを同じパスに配置する**
   ```bash
   # 再生成されたファイルを正しいパスに配置
   mv new_m20_03.png public/images/local-avatars/m20_03.png
   ```

5. **DBやコードのアバターIDは変更不要**（同じファイル名で上書きするため）

## スタイル統一の基準

すべてのアバターは以下の基準を満たすこと：

- **スタイル**: 日本アニメ風フラットイラスト
- **輪郭線**: 太めのダーク輪郭線あり
- **背景**: ベージュピンクの円形背景、白い正方形キャンバス
- **肌色**: 温かみのあるパステルトーン（人物の場合）
- **サイズ**: 2048×2048px（表示時は64〜128pxにリサイズ）
- **テキスト**: なし
- **参照ファイル**: `reference.png`
